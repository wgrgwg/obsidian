# 1. 암호 기초
### 암호 기초 용어
- m : 평문
- Ka(m) : 암호문 (key : Ka)
- m=Kb(Ka(m)) : 복호화

### 암호 깨기
- 두 방법
	- 브루트 포스 : 다해보기
	- 통계적 분석
- known-plaintext attack
	- 일반적으로 많이쓰는 case들 모음집

### 대칭키 암호화
- Symmetric key crpytography (대칭키 암호화)
- 암호화, 복호화 같은 키를 사용
- 단순 암호화 방법
	- 단순히 대체 텍스트로 맵핑
- 좀 더 세련된 암호화 방법
	- 블럭 단위, 싸이클 형태로 구현
	- 조금 더 안전
- DES : Data Encryption Standard
	- 안전한 표준이었음
	- 56비트 암호문, 64비트 평문 입력
	- 3DES : 3가지 다른 키로 3번 암호화
- AES : Advanced Encripytion Standard
	-  DES를 대체하고 표준이 됨
	- 데이터를 128비트 블록으로 처리

### 공개키 (비대칭)
- 공개키
	- 암호화, 모두에게 공개
- 개인키
	- 복호화, 개인만 공유(비공개)
- A키로 암호화 -> B키로 복호화
	- B키로 암호화 -> A키로 복호화
- 단점
	- 대칭키보다 느림
- HTTPS에서 공개키 + 대칭키
	- 공개키 : 키 교환
	- 대칭키 : 데이터 암호화

### 공개키 암호화
- Diffle-Hellman75, RSA78 다른 방식으로 접근
	- 가장 안전한
- 송신자, 수신자가 비밀키를 공유하지 않음

## **RSA 알고리즘**
- 요구사항
	1. K-(K+(m)) = m
	2. 공개키에서 개인키를 계산하는 것은 불가능
- 거꾸로도 가능해야 함
	- key A로 암호화, key B로 복호화
	- key B로 암호화, key A로 복호화

## 디지털(전자) 서명
- 서명한 사람의 인증 용도
- **공개키** 활용
- 절차
	- 문서 일부 **해싱**
	- 해시 정보에 **개인키로 암호화**
	- 수신자는 **공개키로 복호화**
- 상세
	- 송신자는 문서 정보의 해시값 계산, **해시값을 개인키로 암호화**
	- 문서와 **전자서명(암호화된 해시값)**을 같이 전송
	- 수신자는 송신자의 **공개키로 전자서명을 복호화**, 실제 보낸 **문서의 해시값과 비교**
- 해시 함수 알고리즘
	- MD5
	- SHA-1

## 인증서
- 전자서명만으로 송신자의 신우너 확인 불가능
- 공개키+개인키 소유자의 신원 확인
	- 제 3기관의 공인 -> 인증서
- 인증서 내용
	- 신원정보, 공개키, 유효기관, 인증기관정보, 인증기관의 전자서명
- 표준규격 : X.509
- 비대칭키와 인증서 만들기

### X.509
- 인증서 발급시 .der 혹은 .pem 확장자 파일
- 인증서 표준

# 2. IP계층 보안(IPSec)
## IPSec
- IP 패킷에서 암호화, 인증, 무결성
- 두가지 모드
	1. Transport mode 전송 모드
		- IP 패킷의 데이터 부분만 암호화(헤더 제외)
		- end-to-end
	2. Tunnel mode 터널 모드
		- 전체 IP 패킷 암호화
		- network-to-network
		- e.g. VPN


### IPsec Protocols
1. Authentication Header (AH) : 인증헤더
	- 데이터 무결성, 인증 제공
	- 데이터의 기밀성 보장 X
2. Encapsulation Security Protocol (ESP)
	- 데이터 무결성, 인증, 기밀성 제공

### Security Associations (SAs)
- 두 peer 간의 공유 보안 속성을 설정
- IP는 무상태지만, IPsec은 연결 필요!
	- Router 사이에 state를 유지

# 3. 이메일
## 안전한 이메일
### 기밀성
- 송신자
	- 대칭키로 메시지 암호화
	- 공개키로 대칭키 암호화
	- 대칭키로 암호화된 메시지 + 수신자의 공개키로 암호화된 대칭키 전송
	- K<sub>s</sub>(m) + K<sub>b</sub><sup>+</sup>~(K<sub>s</sub>)
- 수신자
	- 개인키로 암호화된 대칭키 복호화
	- 대칭키로 메시지 복호화

### 무결성, 인증
- 무결성, 인증
	- 개인키로 문서의 해시값 암호화
	- 문서와 함께 보냄
- 기밀성
	- 대칭키로 문서 암호화
	- 수신자의 공개키로 대칭키 암호화

### PGP : Pretty Good Privacy
- 데이터를 암호화하고 복호화하는데 사용되는 컴퓨터 프로그램
- Phil Zimmermann
- 전자서명
- 암호화
	- AES256 : 대칭키 알고리즘
- GnuPGP

### S/MIME
- 굉장히 안전한 mail 보안 프로토콜
- MIME
	- multi-purpose internet mail exchange 
	- 이메일 확장 표준
- 암호화 + 디지털 서명
	- 기밀성, 인증, 무결성


# 4. Firewall
## 방화벽
- 보안 관련 프로토콜
- why?
	- port 번호 취약점 공격 방어
	- url 공격 방어
	- 불법 변조/접근 방어
	- 인가된 접근만 허용
- 세가지 타입
	- stateless packet filters
	- stateful packet filters
	- application gateways

### Sateless packet filters
- 각 패킷을 개별적으로 검사(헤더만 검사)
- 패킷의 상태나 연관성 고려 X
	- 규칙을 우회하기 쉬워짐

### Stateful packet filters
- 패킷이 속한 네트워크 연결의 상태를 추적하고 고려
- 패킷이 유효한 연결의 일부인지 아닌지를 확인

### Application Gateways
- 응용 수준에서 트래픽을 필터링, 패킷의 내용도 검사
- 특정 애플리케이션의 사용 제어, 사용자 인증 요구도 가능

### Access Control List : ACL
- 접근제어 리스트(정책)
- Stateful 방법에선 패킷의 상태도 함께 고려됨
- AG에서는 특정 어플 트래픽 제어에도 이용됨

### 공유기 방화벽
- `iptables -nL -v` : 상태 X, 단순한 port 번호만.
- `sudo ufw status`
	- `ufw` : Ubuntu default 방화벽 도구

### 한계
- 복잡해지면 성능상 문제
- 여전히 위협 존재

### Intrusion detection systems : IDS
- 보통은 IP/포트만 검사 
	- 하지만 IDS는 ==**packet payload까지 검사**==
